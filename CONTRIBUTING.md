# Contributing to Guild Boss Timer

Thank you for your interest in contributing to Guild Boss Timer! üéÆ

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a new branch
4. Make your changes
5. Test thoroughly
6. Submit a pull request

## Development Setup

See [QUICKSTART.md](QUICKSTART.md) for setup instructions.

## Code Style

### TypeScript
- Use TypeScript for all new code
- Follow existing naming conventions
- Add type definitions for all functions
- Use interfaces for data structures

### React/Next.js
- Use functional components
- Use hooks for state management
- Server Components by default
- Client Components when needed ('use client')
- Keep components focused and reusable

### Backend
- Use async/await for asynchronous operations
- Parameterized queries for database operations
- Error handling in all routes
- Validate input data

### Styling
- Use Tailwind CSS classes
- Follow existing color scheme
- Ensure mobile responsiveness
- Maintain dark mode support

## Commit Messages

Use clear, descriptive commit messages:

```
feat: Add new boss filtering option
fix: Resolve timer color bug
docs: Update API documentation
refactor: Improve database query performance
```

## Testing

Before submitting:

1. Test all modified features
2. Test on mobile and desktop
3. Verify API endpoints work
4. Check for TypeScript errors
5. Test with different browsers

## Pull Request Process

1. Update README.md if needed
2. Update API.md for API changes
3. Add/update comments in code
4. Ensure no console errors
5. Test deployment process

## Feature Requests

- Open an issue with "Feature Request" label
- Describe the feature clearly
- Explain use cases
- Provide examples if possible

## Bug Reports

Include:
- Description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if applicable
- Environment (OS, browser, versions)

## Areas for Contribution

### High Priority
- Unit tests
- Integration tests
- Performance optimization
- Error handling improvements
- Security enhancements

### Medium Priority
- Additional boss fields
- More filter options
- Export/import functionality
- Notifications/alerts
- Multi-user support

### Nice to Have
- Boss images/icons
- Sound alerts
- Push notifications
- Mobile app
- Discord integration

## Code Review

All submissions require review. We review for:

- Code quality
- Performance
- Security
- User experience
- Documentation

## Questions?

Open an issue or reach out to maintainers.
